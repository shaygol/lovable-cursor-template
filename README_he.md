# Lovable to Cursor: <TITLE>

> [README in English](README.md) | [Original Lovable README](README_lovable.md)

יישום אינטרנט מתקדם בעברית (RTL) שנבנה עם טכנולוגיות הקצה של 2026. הפרויקט מבוסס על תשתית React עוצמתית, מעוצב בסטנדרט פרימיום, ומותאם לעבודה אינטנסיבית עם סוכני AI. המטרה של תבנית זו היא לגשר על הפער שבין בניית אב-טיפוס מהיר לבין פיתוח אפליקציה יציבה, תקנית וניתנת לתחזוקה לאורך זמן.

## מה זו התבנית הזו?

[Lovable](https://lovable.dev) הוא כלי מבוסס AI שמייצר אפליקציות React מלאות תוך דקות - אבל הקוד שנוצר הוא רק נקודת ההתחלה. ברגע שמתחילים לפתח ברצינות ב-Cursor, צריך תשתית מקצועית: חוקים ברורים ל-AI, מדריכי עיצוב, מיפוי של בסיס הנתונים ומבנה תיקיות שהסוכן יודע לנווט בו.

**התבנית הזו עושה בדיוק את זה.** היא מספקת את כל שכבת ה"תודעה" שצריכה לעטוף את הקוד מ-Lovable:

- **`.cursorrules`** — חוקי פיתוח שמונעים מה-AI לעשות טעויות ומכוונים אותו לסטנדרטים גבוהים.
- **`ai-utils/`** — קבצי הקשר (Context) שמאפשרים ל-AI להבין את העיצוב, בסיס הנתונים והאלמנטים הויזואליים של הפרויקט.
- **READMEs מפורטים** — תיעוד דו-לשוני (עברית ואנגלית) שמלווה את המפתח ואת הסוכן כאחד.

## ארכיטקטורת המערכת (The Stack)

הפרויקט נבחר בקפידה כדי לספק ביצועים מקסימליים, תחזוקה קלה וחוויית משתמש חלקה:

- **Framework:** React 18+ מבוסס Vite לביצועים מהירים ו-TypeScript להבטחת איכות הקוד ומניעת שגיאות בזמן פיתוח.
- **Styling:** Tailwind CSS בשילוב עם shadcn/ui ליצירת ממשק משתמש מודולרי, אחיד ונגיש.
- **Icons:** Lucide React - ספריית אייקונים וקטורית, קלה וגמישה להתאמה אישית.
- **Data:** TanStack Query לניהול המצב מול Supabase (Backend-as-a-Service), הכולל ניהול Cache וסנכרון נתונים אוטומטי.
- **Forms:** ניהול טפסים מורכבים באמצעות React Hook Form ואימות נתונים קשיח (Schema Validation) עם Zod.
- **Charts:** Recharts להצגת נתונים ויזואלית, דינמית ורספונסיבית.
- **Animations:** Framer Motion לאנימציות ממשק עדינות, מעברים חלקים וחוויית משתמש יוקרתית.
- **Dates:** date-fns לניהול תאריכים מורכב, חישובים ותמיכה מלאה בלוקאליזציה עברית.

## RTL & Hebrew (עברית)

האפליקציה מתוכננת מהיסוד לתמיכה מלאה בכתיבה מימין לשמאל (Right-to-Left):

- **Logical Properties:** שימוש בלעדי במאפיינים כמו `ms-*` (Margin Start) ו-`pe-*` (Padding End) במקום צדדים קבועים (Left/Right) כדי להבטיח גמישות והתאמה אוטומטית לכל כיווני הכתיבה.
- **Typography:** שימוש בפונטים מודרניים המותאמים לעברית דיגיטלית כמו 'Heebo' או 'Assistant', תוך שמירה על היררכיית משקלים נכונה (Bold לכותרות, Regular לטקסט רץ).
- **Localization:** כל הטקסטים המוצגים למשתמש, הודעות השגיאה והלייבלים בטפסים כתובים בעברית תקנית ומקצועית.

## קובץ החוקים (.cursorrules)

קובץ ה-`.cursorrules` הוא הלב הפועם של הפיתוח ב-Cursor. הוא מגדיר ל-AI את ה"אישיות" המקצועית שלו ומבטיח שהקוד שייכתב יהיה איכותי ואחיד:

- **אכיפת סטנדרטים:** הקובץ מונע מה-AI להשתמש בקיצורי דרך מסוכנים (כמו טיפוס `any`), ומחייב כתיבה פונקציונלית, נקייה ומודולרית.
- **ניהול RTL אוטומטי:** ה-AI מקבל הוראה מפורשת להשתמש ב-Logical Properties, מה שמונע שבירה של הממשק בעברית כאשר מוסיפים רכיבים חדשים.
- **מניעת ניחושים:** הקובץ יוצר פרוטוקול בטיחות - אם המידע על בסיס הנתונים חסר, ה-AI מחויב לעצור ולבקש הבהרות במקום "להמציא" שמות של טבלאות.
- **המשכיות מ-Lovable:** ה-AI מזהה באופן אוטומטי פרויקטים שהגיעו מ-Lovable ומכבד את מוסכמות הכתיבה שלהם (כמו ה-Tagger וה-Aliases), מה שמונע קונפליקטים בקוד.

## פיתוח מבוסס AI (AI-Assisted Development)

תיקיית `ai-utils/` היא המרכז שבו נשמר ההקשר (Context) של הפרויקט. היא מבטיחה שה-AI יבין את המערכת לעומק:

- **`ui-style.md`** — ה-DNA העיצובי: מגדיר איך נראה לוח בקרה יוקרתי, איך מתנהגים טפסים חכמים ומהו הריווח הנכון בין כרטיסי מידע. זהו ה-"Style Guide" שמנחה את ה-AI לשמור על מראה ה-Premium.
- **`visual-assets.md`** — עקביות ויזואלית: מבטיח שסטטוס "פעיל" תמיד יהיה ירוק עם אותו אייקון, ושסרגל הניווט ישתמש באותם סמלים בכל דף חדש שנוצר.
- **`db-schema.md`** — מפת הדרכים לשאילתות: מאפשר ל-AI לכתוב קוד צד-שרת (Supabase) מדויק על ידי הבנת הקשרים בין הטבלאות (Foreign Keys) וסוגי הנתונים.

## זיהוי Lovable

הפרויקט כולל מנגנון זיהוי חכם המזהה אם המקור נוצר ב-Lovable. במקרה כזה, ה-AI מונחה לכבד את ה-`lovable-tagger`, את מבנה ה-Components הקיים ואת השימוש ב-`@` כנתיב קיצור לתיקיית ה-`src`, מה ששומר על אינטגרציה מושלמת בין הכלי העיצובי לקוד המקומי.
